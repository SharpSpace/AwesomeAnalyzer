# JJ0006
Remove async and await when awaiting completion tasks.

This rule warns when `async`/`await` is used with completion tasks like `Task.CompletedTask`, `Task.FromResult()`, or `new ValueTask()` where no actual asynchronous work is happening.

## Violation
```C#
using System.Threading.Tasks;

sealed class Program
{
    public async Task Method()
    {
        await Task.CompletedTask.ConfigureAwait(false);
    }
}
```

## Fix
```C#
using System.Threading.Tasks;

sealed class Program
{
    public Task Method()
    {
        return Task.CompletedTask;
    }
}
```

## Notes
This rule only triggers for completion tasks where nothing actually happens:
- `Task.CompletedTask`
- `Task.FromResult(...)`
- `new ValueTask(...)`
- `Task.Delay(0)`

For methods that await actual async operations, the rule will not trigger.
